# Hello World

Well, the site is live! It took a bit longer than initially anticipated, but it has all the signs of a planned project:

* Create a repo on GitHub and do nothing with it for a few days - check
* Browse all the Static Site Generators and try to sniff out the most appropriate one - check
* Try out a few SSGs and decide they are all unnecessarily confusing (which means make no significant effort to understand the concepts of any one framework) - check
* Decide an SSG is an overkill anyway for a simple personal site, just HTML/CSS the whole thing - check
* Use Claude Code to eventually build a tiny SSG (couple of templates and a build script) because they do solve common issues - check
* Feel good about having gone full circle on the topic - check

I try to be funny writing the above, but I did end up going full circle indeed. The difference between my little implementation and something robust like Hugo or Jekyll is that I'm only doing it for myself, so I don't need to support multiple use cases, themes and so on. One and done, until I need a new feature.

Nevertheless, it's been fun exploring this topic. I recently came across a fascinating website called [512KB Club](https://512kb.club/) which lists websites that are no more than 512KB in total size and I was inspired by it. I have been saying for a few years now we need to go back to the HTML 1.1 days, conceptually speaking. Yes online shopping is convenient, and online learning is magnificent, but not all websites need to be a ridiculously large collection of frameworks and libraries and 10MP images. I usually use the example of professors websites in university servers. They are typically pure HTML, perhaps with a table that matches the semester, or another table that links to the courses they teach, but they contain essential material. In the end the content is what matters.

Which brings me to another thing I've been thinking. Modern browsers are so feature-packed, we are getting in their way the more we overdesign our websites. Safari for example comes with Reader Mode and Firefox is one plug-in away from the same feature. Readers can simply toggle that whenever they want to read long form content using nice (nicer) fonts with appropriate spacing for the screen they are using. No need to pull external fonts on every web landing. The same goes for Light/Dark mode. It is just a CSS setting that follows the browser/system.

In any case, I am finding that the more I think about these topics (technology, usability, modernism, minimalism) the better it is to write them down than keep stirring them in my head. I hope to make use of this new space to do just that.

Cheers.
