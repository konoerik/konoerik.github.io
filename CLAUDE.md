# CLAUDE.md

## Project Overview
Personal website at konoerik.github.io, hosted on GitHub Pages. Contains static pages
(landing, about, projects with subpages) and a dated blog. Built with raw HTML/CSS;
Python scripts handle the mechanical build step. No frameworks, no static site generators.

## Tech Stack
- HTML/CSS (no JS framework, no CSS framework)
- Python 3 build scripts (minimal dependencies — `markdown`, `pyyaml`; managed via `uv`)
- GitHub Actions for automated deploy on push to main

## Key Conventions
- Source files live in `src/`; build output goes to `site/` (git-ignored)
- Template: `src/_templates/base.html` — `{{ title }}`, `{{ description }}`, `{{ content }}`, `{{ root }}` placeholders
- Page frontmatter: HTML comment block at top of each source file (`<!-- key: value -->`)
- Blog posts: `src/blog/posts/YYYY-MM-DD-slug.html` — blog index is auto-generated by build
- Project subpages: `src/projects/project-name/index.html`

## Development Workflow
```bash
# build:    python3 admin.py build  (or: make build)
# new post: make new-post TITLE=Post Title
# serve:    make serve  (builds then serves at localhost:8000)
```

## Behavior Rules
- Never add JS frameworks, CSS frameworks, or pip dependencies to the build
- Do not modify files in site/ directly — always edit src/ and rebuild
- Keep admin.py dependencies minimal and readable top to bottom
